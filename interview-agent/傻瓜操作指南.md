4. 点 **创建** → 立刻复制那一长串 `sk-xxxx...` 到记事本
5. ⚠️ **这个 key 只显示一次，离开页面就看不到了！**

## 第二步：配置 Supabase（数据库）

### 2.1 创建项目

1. 打开 [supabase.com](https://supabase.com)
2. 点 **Start your project** → 用 GitHub 登录
3. 点 **New project**
4. 填写（其他全部默认）：

| 字段 | 填什么 |
|:-----|:-----|
| Name | `interview-agent` |
| Database Password | 自己设一个密码，**记下来** |
| Region | 选 `ap-southeast-1` (新加坡，离中国近) |

5. 点 **Create new project** → 等 1-2 分钟（页面显示转圈）

### 2.2 记录连接信息

1. 左侧点 **⚙️ Settings**（齿轮图标）
2. 点左侧 **API**
3. 你会看到两个值，**复制到记事本**：

| 字段 | 在哪 | 长什么样 |
|:-----|:-----|:-----|
| **Project URL** | `Config` → `URL` | `https://xxxxxxxxxxxx.supabase.co` |
| **anon public key** | `Project API keys` → `anon public` | `eyJhbGciOi...` 很长一串 |

### 2.3 创建数据库表

1. 左侧点 **SQL Editor**（终端图标）
2. 点 **New query**
3. **把下面所有 SQL 全部复制粘贴进去**：

```sql
-- 启用向量扩展
CREATE EXTENSION IF NOT EXISTS vector;

-- 文档表
CREATE TABLE IF NOT EXISTS documents (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    repo TEXT NOT NULL,
    path TEXT NOT NULL,
    content TEXT NOT NULL,
    embedding vector(512),
    level TEXT NOT NULL CHECK (level IN ('project', 'architecture', 'code', 'history')),
    language TEXT DEFAULT '',
    start_line INTEGER DEFAULT 0,
    end_line INTEGER DEFAULT 0,
    content_hash TEXT,
    metadata JSONB DEFAULT '{}',
    created_at TIMESTAMPTZ DEFAULT now(),
    updated_at TIMESTAMPTZ DEFAULT now()
);

-- 索引
CREATE INDEX IF NOT EXISTS idx_documents_repo ON documents(repo);
CREATE INDEX IF NOT EXISTS idx_documents_level ON documents(level);

-- 关键词搜索用的索引
CREATE INDEX IF NOT EXISTS idx_documents_content ON documents USING gin (to_tsvector('simple', content));

-- RPC: 向量搜索
CREATE OR REPLACE FUNCTION match_documents(
    query_embedding vector(512),
    match_threshold float DEFAULT 0.5,
    match_count int DEFAULT 5
)
RETURNS TABLE(
    id UUID, repo TEXT, path TEXT, content TEXT,
    level TEXT, language TEXT, metadata JSONB, score float
)
LANGUAGE plpgsql AS $$
BEGIN
    RETURN QUERY
    SELECT d.id, d.repo, d.path, d.content, d.level,
           d.language, d.metadata,
           1 - (d.embedding <=> query_embedding) AS score
    FROM documents d
    WHERE d.embedding IS NOT NULL
      AND 1 - (d.embedding <=> query_embedding) > match_threshold
    ORDER BY d.embedding <=> query_embedding
    LIMIT match_count;
END;
$$;

-- RPC: 项目列表
CREATE OR REPLACE FUNCTION list_projects()
RETURNS TABLE(
    name TEXT, description TEXT, language TEXT,
    stars INTEGER, url TEXT, topics TEXT[], last_updated TEXT
)
LANGUAGE plpgsql AS $$
BEGIN
    RETURN QUERY
    SELECT DISTINCT ON (d.repo)
        d.repo AS name,
        COALESCE(d.metadata->>'description', '') AS description,
        COALESCE(d.metadata->>'primary_language', '') AS language,
        COALESCE((d.metadata->>'stars')::int, 0) AS stars,
        COALESCE(d.metadata->>'html_url', '') AS url,
        COALESCE(ARRAY(SELECT jsonb_array_elements_text(d.metadata->'topics')), ARRAY[]::TEXT[]) AS topics,
        COALESCE(d.metadata->>'updated_at', '') AS last_updated
    FROM documents d
    WHERE d.level = 'project'
    ORDER BY d.repo, d.updated_at DESC;
END;
$$;

-- RPC: 索引统计
CREATE OR REPLACE FUNCTION index_stats()
RETURNS TABLE(total_docs BIGINT, total_repos BIGINT, last_indexed_at TEXT)
LANGUAGE plpgsql AS $$
BEGIN
    RETURN QUERY
    SELECT COUNT(*)::BIGINT, COUNT(DISTINCT repo)::BIGINT,
           to_char(MAX(updated_at), 'YYYY-MM-DD HH24:MI:SS')
    FROM documents;
END;
$$;
```

4. 点右下角 **Run**（或按 `Ctrl+Enter`）
5. 看到绿色提示 `Success` 就 OK 了

> ✅ **验证**：左侧点 **Table Editor**，能看到 `documents` 表，说明成功。

## 第三步：配置 Cloudflare Workers

### 3.1 获取 Account ID

1. 打开 [dash.cloudflare.com](https://dash.cloudflare.com) → 注册/登录
2. 登录后，右侧栏会看到 **Account ID**（32 位 hex 字符串）
3. **复制到记事本**，长这样：`a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6`

### 3.2 创建 API Token

1. 右上角点头像 → **My Profile**
2. 左侧点 **API Tokens**
3. 点 **Create Token**
4. 找到 **Edit Cloudflare Workers** 模板，点 **Use template**
5. 两个下拉框都选你的账号
6. 点 **Continue to summary** → **Create Token**
7. **立刻复制**那一长串 token 到记事本
8. ⚠️ **这个也只显示一次！**

## 第四步：设置 GitHub Secrets

### 4.1 找到设置位置

1. 打开你的 GitHub 仓库：`github.com/Rcardo/interview-agent`
2. 点顶部 **Settings**
3. 左侧点 **Secrets and variables** → **Actions**
4. 点 **New repository secret**

### 4.2 逐个添加（每加一个点一次 Add）

| Name（名称） | Secret（值） | 从哪来 |
|:----|:-----|:-----|
| `DEEPSEEK_API_KEY` | `sk-xxxx...` | DeepSeek 第一步拿到的 key |
| `SUPABASE_URL` | `https://xxxx.supabase.co` | Supabase 第二步的 Project URL |
| `SUPABASE_ANON_KEY` | `eyJhbGciOi...` | Supabase 第二步的 anon key |
| `CLOUDFLARE_API_TOKEN` | 一长串 | Cloudflare 第三步的 API Token |
| `CLOUDFLARE_ACCOUNT_ID` | `a1b2c3...` | Cloudflare 第三步的 Account ID |

> 加完应该显示 **5 个 Secrets**。

## 第六步：配置 Worker 环境变量（关键！）

GitHub Actions 部署了 Worker 代码到 Cloudflare，但不会自动设 Secret。需要手动设一次：

1. 打开 [dash.cloudflare.com](https://dash.cloudflare.com)
2. 左侧点 **Workers & Pages**
3. 找到 `interview-agent-api`，点进去
4. 点顶部 **Settings** → **Variables and Secrets**
5. 逐个添加：

| 类型 | Name | Value |
|:-----|:-----|:-----|
| **Secret** | `DEEPSEEK_API_KEY` | `sk-xxxx...` |
| **Secret** | `SUPABASE_URL` | `https://xxxx.supabase.co` |
| **Secret** | `SUPABASE_ANON_KEY` | `eyJhbGciOi...` |
| **Plain text** | `GITHUB_USERNAME` | `Rcardo` |
| **Plain text** | `CANDIDATE_NAME` | `向荣鑫` |
| **Plain text** | `CANDIDATE_TITLE` | `软件工程师` |

6. 点 **Save** → 点 **Deploy**（重新部署生效）

> 加完 Secret 后 Workers 会自动重新部署，等 30 秒就好。

## 🔧 常见问题排查

### Q: 页面打开了但发消息没反应？
**A:** 检查 Worker URL 是否正确配到了 GitHub Variables 的 `VITE_API_BASE`。打开浏览器 F12 → Console，看有没有红色报错，报 404 就是 URL 不对。

### Q: Worker 报 500 错误？
**A:** 去 Cloudflare Workers → 点 `interview-agent-api` → 点 **Logs**，看具体错误。通常是 Secret 没配全，回第六步检查。

### Q: 索引跑完但搜索不到项目？
**A:** 去 Supabase → **Table Editor** → 打开 `documents` 表，看有没有数据。如果为空，说明索引没成功，去 Actions 看 Index GitHub Repositories 的日志。

### Q: DeepSeek 返回乱码或空白？
**A:** 检查 `DEEPSEEK_API_KEY` 是否正确（在 Cloudflare Workers Settings 里）。确保 key 没有复制多余的空格。

### Q: 回答和我项目没关系？
**A:** 因为当前用的是**关键词检索**而非向量检索。如果后续配了 `OPENAI_API_KEY`，搜索引擎会自动升级为更精准的向量检索。
